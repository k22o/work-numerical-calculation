# 数値計算の基礎概念

## 固定小数点と浮動小数点

- 固定小数点
    - 小数点の位置をあらかじめ固定して数値を表現する方式
    - 整数部と小数部のビット数が決まっているため、ハードウェアが単純になる
    - 一方で表現できる数値の範囲と精度が固定される
- 浮動小数点
    - 符号部・指数部・仮数部の3要素で数値を表現する方式
    - IEEE 754 規格で定義されており、非常に広い範囲の数値を表現できる
- マシンイプシロン
    - 浮動小数点数において `1.0 + ε ≠ 1.0` となる最小の正の数 ε のこと。計算機が区別できる最小の数値間隔を示す精度の指標。NumPy では `numpy.finfo(float).eps` で取得できる（倍精度で約 2.22e-16）。

### 単精度と倍精度

| | 単精度 (float32) | 倍精度 (float64) |
|---|---|---|
| ビット数 | 32 bit（符号1・指数8・仮数23） | 64 bit（符号1・指数11・仮数52） |
| 有効桁数 | 約7桁 | 約15〜16桁 |
| マシンイプシロン | 約 1.19e-7 | 約 2.22e-16 |
| 用途 | 速度・メモリ優先（機械学習など） | 科学技術計算の標準 |

## 誤差について

1. **打ち切り誤差（truncation error）**
    - 本来は無限回必要な計算（級数展開・積分など）を有限の項数・ステップで打ち切ることで生じる誤差。アルゴリズムの近似に起因する。

2. **丸め誤差（rounding error）**
    - 浮動小数点数で正確に表現できない数値を最も近い表現可能な値に丸めることで生じる誤差。計算を繰り返すと蓄積する（累積誤差）。

3. **桁落ち（cancellation error）**
    - 値がほぼ等しい2数の減算を行うと、有効桁数が大幅に失われる現象。例: `1.000001 - 1.000000 = 0.000001`（有効桁が1桁に激減）。

4. **情報落ち（loss of trailing digits）**
    - 絶対値の大きく異なる2数を加減算するとき、絶対値の小さい数の下位ビットが切り捨てられて情報が失われる現象。例: 大きな数に非常に小さな数を加算しても結果が変わらない場合がある。

## 絶対誤差と相対誤差

真値を $x$、近似値を $\tilde{x}$ とすると：

- **絶対誤差**
    - $|x - \tilde{x}|$ で表される誤差の大きさ。単位はそのままなので、値のスケールに依存する。
- **相対誤差**
    - $\dfrac{|x - \tilde{x}|}{|x|}$ で表される誤差の割合。スケールに依存しないため、異なる大きさの数値の精度比較に適している。
